# Furhat-Robotics-Markers-Via-Lab-Streaming-Layer-LSL-
Furhat Robotics skill in Kotlin for a dice game interaction within a multimodal experimental set-up. The robot prompts dice rolls, collects and logs results, and sends event markers via Lab Streaming Layer through a Python bridge for synchronized data recording.

# Description

This repository contains a custom interaction skill developed for the Furhat Robotics platform in collaboration with master’s student Phelim Korving. The skill was implemented in Kotlin using IntelliJ IDEA and extends the existing ComplimentBot example from the Furhat SDK.

The skill allows the robot to run an **interactive dice game** with a human participant. During the interaction, the robot instructs the participant to roll dice, asks them to report the results, and logs the reported outcomes. All responses and relevant interaction events are saved for later analysis.

To enable synchronization with physiological recording systems, the skill sends event markers via **Lab Streaming Layer (LSL)**. Communication with LSL is handled through a bridge script written in Python. The bridge receives signals from the Furhat skill and forwards them as LSL streams to external acquisition devices.

The system was designed to work with both the NIRx fNIRS acquisition system and the BIOPAC MP160. Because the two devices require different marker formats, the Python bridge initializes and manages two separate LSL streams:
 - The NIRx stream receives discrete numeric markers corresponding to specific experimental events.
 - The BIOPAC stream receives a continuous time-series marker signal, which is generated and maintained by the Python bridge.

This architecture enables synchronized multimodal recordings during the human–robot interaction experiment.

# Usage Notes
## LSL setup
Install and configure Lab Streaming Layer according to the official documentation of Lab Streaming Layer before running the system. Event markers generated by the robot interaction will be transmitted through LSL and recorded alongside the physiological data streams.

## Start order
The Python bridge must be started before launching the Furhat skill. The bridge initializes the LSL streams required by the acquisition devices. If you start the skill before staring the bridge, it will simply not send markers to your device without throwing any error.
